The Graphics Engine

Built on the AWT Canvas architecture, Samarbhumi uses a Triple-Buffered render pipeline to achieve smooth 60 FPS performance even on integrated hardware.

  • Dynamic Scaling: Resolutions range from 720p to 4K with auto-aspect ratio preservation.
  • Micro-Optimization: Frame-independent physics with sub-pixel collision detection.

Robust OOP Architecture

The codebase demonstrates professional-grade Object Oriented Programming to handle complex game states and entity interactions.

  • Inheritance: Deep hierarchies for Entity → Player → OnlinePlayer.
  • Polymorphism: HUDRenderer and EffectGenerator handle diverse visual data through unified interfaces.
  • Encapsulation: Modular logic for Profiling, Store, and Networking layers.

Real-Time Networking

Implementing reliable multiplayer in Java Sockets requires sophisticated synchronization. Samarbhumi uses a Lockstep Protocol.

  • Determinism: Each frame only progresses once all client inputs are received.
  • Watchdog System: Auto-resynchronization and 3-second timeout protection.
  • Compression: Bit-packed input arrays to minimize bandwidth usage.
