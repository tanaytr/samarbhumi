Project Overview

Samarbhumi is a fast-paced combat arena game that focuses on fluid movement and high-fidelity networking. It combines classic arcade mechanics with modern technical standards.

  • Zero Library Dependency: Built without external engines (Unity/Godot) or libraries.
  • Native Graphics: High-speed rendering using Java AWT and Swing BufferStrategy.
  • Real-time Networking: Custom socket protocol for multiplayer sync.
  • Comprehensive Progression: Integrated profile, leveling, and store systems.

Combat & Physics

We implemented a frame-independent physics engine to ensure consistent gameplay across varying hardware speeds.

  • Pixel-Perfect Collision: AABB (Axis-Aligned Bounding Box) resolution for players and tiles.
  • Advanced Movement: Double jumping, jetpack thrust with fuel management, and crouching.
  • Ballistics: Dynamic projectiles with range, spread, and gravity-affected grenades.
  • Entity System: Modular entity management for bullets, pickups, and particles.

Object-Oriented Design

The project follows strict OOP principles to manage complexity and ensure maintainability.

  • Inheritance: Base Entity class for all moving objects (Players, Projectiles).
  • Polymorphism: Unified Renderer interfaces for diverse visual components.
  • Encapsulation: Modular controllers for AI, Input, Audio, and Networking.
  • Design Patterns: Factory (Maps), Singleton (Audio), and Command (Inputs).

Multiplayer Architecture

An custom-built Lockstep Synchronization protocol handles real-time online play through a dedicated Relay Server.

  • Deterministic Simulation: Each frame only advances once all player inputs are received.
  • Input Masking: Bit-packed byte arrays to minimize network latency and bandwidth.
  • Relay Server: Scalable architecture hosted on cloud infrastructure for username-based lobbies.
  • Watchdog Timer: Sophisticated timeout logic to handle desync and disconnection gracefully.

Dynamic Level Design

Four distinct maps designed with tactical variety, each featuring custom tile logic and aesthetic themes.

  • Warzone Alpha: Open field with grass platforms and water trenches.
  • Jungle Ruins: Vertical stone structures with dense ladder networks.
  • Steel Fortress: Industrial bunker with tight corridors and symmetric flow.
  • City Ruins: Urban rooftops with fire escapes and glowing cityscapes.

Maps are represented as bit-tile arrays with integrated collision metadata.

Weaponry & HUD

A diverse arsenal of 7+ weapons, each with unique firing logic, sound FX, and particle trails.

  • Assault Rifle: High fire rate, balanced spread.
  • Rocket Launcher: Low speed, massive splash damage.
  • Dual Pistols: Fast-draw close range dominance.
  • Sniper: Instant velocity with high damage output.

The HUD features real-time killfeeds, ammo tracking, and health indicators using custom vector graphics.

v4.0 Performance Tuning

The latest iteration focuses on "High-Stability" infrastructure to eliminate common Java Swing bottlenecks.

  • Non-Blocking I/O: Handled network handshakes in dedicated threads to prevent UI freezes.
  • Input Queueing: Concurrent click registration to handle rapid user interactions.
  • Optimized Loops: Fixed-timestep game loop for deterministic physics resolution.
  • Clean Disposal: Proper socket closing and thread termination logic.

User Experience (UX)

The game provides a premium experience through polished UI and cinematic audio.

  • Themed Audio: Dynamic music switching between menu, battle, and victory.
  • Responsive Design: Fluid layout systems that adapt to resolution changes.
  • Visual Polish: Particles for explosions, shell casings, and jet trails.
  • Player Profile: Persistent XP, coin rewards, and skin unlock store.

Conclusion

Samarbhumi demonstrates that high-performance interactive media is possible within the native Java ecosystem when leveraging efficient algorithms and robust OOP design.
